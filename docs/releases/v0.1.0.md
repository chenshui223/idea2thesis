# idea2thesis v0.1.0

First public local-deployment release.

## Highlights

- Local single-user Web application for thesis-project generation from a `.docx` graduation design brief.
- Multi-agent execution chain with advisor, coder, writer, reviewer agents, code evaluation, and document checks.
- Persistent history workbench with job detail, event timeline, rerun, soft delete, and workspace ZIP export.
- Generated artifact preview support, including generated `.docx` thesis draft preview.
- Chinese-first UI by default, with one-click English toggle in the top-right corner.
- Open-source onboarding documents now included: README, screenshots, CONTRIBUTING guide, CHANGELOG, Apache 2.0 license.

## Local Deployment

```bash
git clone https://github.com/chenshui223/idea2thesis.git
cd idea2thesis
bash scripts/bootstrap.sh
bash scripts/dev.sh
```

Then open:

- Frontend: `http://127.0.0.1:5173`
- Backend API: `http://127.0.0.1:8000`

## Runtime Requirements

- Python `3.12+`
- Node.js `20+`
- An OpenAI-compatible API endpoint
- Your own `API Key`, `Base URL`, and model configuration

## Security Boundary

- Runtime `API Key` values are intentionally not persisted.
- Local runtime state under `jobs/` and `.idea2thesis/` is git-ignored.
- This project is designed for local use, not hosted multi-user SaaS deployment.

## Known Limitations In v0.1.0

- No automatic multi-round repair loop yet.
- Review failures and blocked outputs still require manual inspection and rerun.
- This release does not include a GitHub-hosted release asset bundle.
